# Discuss AI 翻译 (Discuss AI Translate)

**Odoo Discuss 的 AI 消息翻译 —— 覆盖内部频道、WhatsApp、记录聊天。**

跨语言沟通从此轻松：按需翻译收到的消息；或者用自己的母语打字,
在发送前即时预览对方语言的译文。

---

## 主要功能

- **收消息按需翻译**
  在任意消息上点击三点菜单 → **AI Translate**,选择目标语言,
  译文直接显示在气泡下方。原文保留,折叠面板即可隐藏译文。

- **用母语写,对方语言发**
  在输入框开启翻译模式,用中文/阿拉伯语/其他语言打字,
  输入过程中实时看到英文(或任意目标语言)预览。点 **Use**
  发送译文。已发送消息下方会显示 "View original" 链接,
  随时找回自己写的原文。

- **所有聊天场景通用**
  内部频道、WhatsApp 对话、记录聊天器、私聊、在线客服。

- **原生 Odoo AI 集成**
  使用 Odoo 19 内置的 AI 框架,无需额外配置。配置一个
  **OpenAI** 或 **Google Gemini** API Key 即可运行。如果两个都
  配置,一个服务商失败会自动切换另一个。

- **成本控制**
  利用 Odoo 原生的 `mail.message.translation` 表做消息级缓存,
  多人打开同一条消息不会重复调用 API。

- **隐私友好**
  除了被翻译的消息内容外,没有其他数据离开 Odoo 实例。
  翻译请求直接发往你配置的 AI 服务商。

## 系统要求

- Odoo 19 企业版
- 原生 `ai` 和 `ai_app` 模块(企业版自带)
- 在 *设置 → AI* 中配置 **OpenAI** 或 **Google Gemini** API Key

---

## 安装配置

1. 安装此模块
2. 进入 **设置 → AI**,至少添加一个服务商 API Key
3. 同一页面选择默认翻译模型(默认 `gpt-5-mini` —— 快且省钱)

完成。每条消息上立刻出现 *AI Translate* 按钮,输入框上也会
出现翻译开关。

### 高级配置(可选)

少数长尾参数通过系统参数(*技术 → 参数 → 系统参数*)调整,
没有设置界面,默认值覆盖 99% 的使用场景:

| 参数                                       | 默认值  | 用途                       |
|--------------------------------------------|---------|----------------------------|
| `suite_ai_translate.max_input_chars`        | `8000`  | 单条消息翻译的字符数上限   |
| `suite_ai_translate.temperature`            | `0.2`   | LLM 采样温度               |

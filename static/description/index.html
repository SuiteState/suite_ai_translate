Discuss AI Translate

AI-powered translation for Discuss channels, WhatsApp, and chatter

Break the language barrier in every conversation

Multilingual teams and cross-border customer conversations no longer need a copy-paste detour through an external translator. Discuss AI Translate brings one-click translation directly into your Discuss channels, WhatsApp conversations, and record chatters — powered by the AI provider you already use in Odoo.

What it does

Translate incoming messages on demand

Open any message's three-dot menu and click AI Translate. The translated text appears inline below the original, with a language picker supporting over 100 languages. Your last-used target language is remembered.

Translate before you send

Type a draft in your own language, click AI Translate above the composer, then send the translated version with a single click. The original text is archived on the message for internal reference via View original.

Works everywhere Discuss works

Internal channels, direct messages, record chatter (CRM, Sales, Helpdesk, anywhere mail.thread is mounted), and WhatsApp conversations — one module, consistent UX.

Automatic provider failover

Configure a primary model (e.g. OpenAI) and a failover model (e.g. Google Gemini). If the primary provider fails — network error, rate limit, invalid key — translation automatically retries on the failover. No dropped messages.

Translation caching

Results are cached per message + target language in Odoo's native translation table (mail.message.translation). Re-opening a previously translated message is instant and costs no API credits. Cache auto-vacuums after 2 weeks.

Uses Odoo's native AI framework

No extra API key beyond what Odoo's ai module already needs. No third-party SaaS account. The AI provider and key live in Odoo's standard AI settings — this module just consumes them.

Data transmission & privacy

This module transmits message text to a third-party AI provider (OpenAI or Google Gemini, depending on your configuration) over HTTPS for the sole purpose of translation. Please read this section carefully and ensure compliance with your organization's data handling policies before enabling the feature.

What is sent

  • The plain-text body of the message or composer draft selected for translation.
  • The target language code (e.g. en, ar).

What is NOT sent

  • No author names, recipient names, partner records, email addresses, phone numbers, or any other personally-identifiable metadata.
  • No attachments, images, or files.
  • No database records, CRM context, sales data, or invoice data.
  • No analytics, telemetry, or usage tracking to any party other than the AI provider you have configured.

Who receives the data

Whichever provider your administrator has configured in Odoo's AI settings — OpenAI (ChatGPT) and/or Google Gemini. Transmitted text is subject to that provider's terms of service, data retention policies, and applicable privacy law. Please review:

  • OpenAI: https://openai.com/policies/
  • Google Gemini: https://ai.google.dev/terms

Opt-in by design

The feature is disabled by default. An administrator must explicitly enable Enable Discuss AI Translate in Settings → General Settings → AI → Discuss AI Translate before any text is transmitted. Translation RPC endpoints refuse to call any AI provider until this opt-in is granted.

Local caching

Translated results are cached in your own Odoo database (table mail.message.translation). Cache entries are not transmitted anywhere and are auto-removed by Odoo's native cleanup after 2 weeks of non-access.

Requirements

  • Odoo 19 Enterprise. Depends on the ai and ai_app modules, which are part of the Enterprise edition.
  • An OpenAI or Google Gemini API key configured in Settings → General Settings → AI. The module does not bundle or manage its own keys.
  • Network egress from the Odoo server to api.openai.com and/or generativelanguage.googleapis.com.
  • Optional — for WhatsApp translation: Odoo's native whatsapp module installed and configured with a valid Meta WhatsApp Business Account. This module does NOT integrate with the WhatsApp Business API itself — it only recognizes WhatsApp conversations that Odoo's native module has already set up, and adds translation capability to them. Setting up WhatsApp in Odoo (Meta Business Account, phone number verification, template approval, etc.) is entirely handled by the native whatsapp module and is not in scope of this app. If you don't use WhatsApp, everything else works without it.

Supported languages

Over 100 languages, covering the major world business languages and most regional ones: English, Arabic, Chinese (Simplified and Traditional), Spanish, Hindi, Portuguese, Bengali, Russian, Japanese, German, French, Urdu, Indonesian, Italian, Turkish, Korean, Vietnamese, Persian, Polish, Thai, Dutch, Ukrainian, Hebrew, Swahili, and many more. Because translation is performed by the LLM, adding a language to the picker is a UI-level change — the translation quality for a given language pair depends on the AI provider.

Right-to-left languages (Arabic, Persian, Urdu, Hebrew, Pashto, Kurdish, Yiddish) render with correct dir="rtl" in the translation preview and panel.

Configuration

All configuration lives in Settings → General Settings → AI → Discuss AI Translate:

  • Enable Discuss AI Translate — master opt-in switch (OFF by default).
  • Translation Model — primary model used for every translation request.
  • Failover Model — automatic fallback when the primary provider fails. For real resilience, choose a model from a different provider.

Advanced: sampling temperature can be tuned via the system parameter suite_ai_translate.temperature (default 0.2). Raising it makes translations slightly more varied; most users should not need to touch this. The per-message length cap can be adjusted via suite_ai_translate.max_input_chars (default 8000).

# Discuss AI 翻译 (Discuss AI Translate)

Odoo Discuss、WhatsApp、记录聊天的双向 AI 消息翻译。

翻译支持两个方向:

- **接收方向** —— 按需翻译收到的消息以便阅读。
- **发送方向** —— 在发送前翻译输入框中的草稿;原文存档在已发送消息上。

翻译由 Odoo 原生 `ai` 模块中配置的 AI 服务商执行(OpenAI 或
Google Gemini)。本模块不需要单独的 API Key 或 SaaS 账号。

---

## 功能

- **发送消息翻译**
  在输入框中用任意源语言打字,点击输入框上方的 *AI Translate*,
  然后发送译文。原文草稿存档在已发送消息上,可通过 *View original*
  访问。

- **接收消息翻译**
  打开消息的三点菜单选择 *AI Translate*。译文显示在原文下方,
  语言选择器支持超过 100 种语言。每个用户的上次目标语言会被记住。

- **聊天界面覆盖**
  Discuss 频道和私聊、浮动聊天窗口、WhatsApp 对话、邮件 composer、
  任意业务记录的聊天器(CRM、销售、服务台等任何挂载 `mail.thread`
  的模型)、在线客服后台视图。

- **服务商容错切换**
  可配置一个主模型和一个备用模型。如果主服务商调用失败(网络错误、
  限流、API Key 无效),请求会自动重试备用模型。

- **翻译缓存**
  翻译结果按消息 + 目标语言缓存在 Odoo 原生
  `mail.message.translation` 表中。缓存条目在 2 周未访问后自动清理。

- **隐私设计**
  只传输待翻译的文本和目标语言代码。不传输作者/收件人姓名、
  附件、记录上下文。功能默认关闭,需要管理员手动启用。

---

## 适用范围

### 支持

- Discuss 内部频道和私聊
- 浮动聊天窗口(从消息菜单弹出的右下角小窗)
- WhatsApp 对话(需先安装并配置 Odoo 原生 `whatsapp` 模块)
- 邮件 composer (`mail.compose.message`)
- 任意业务记录的聊天器(CRM、销售、服务台、任务,
  任何挂载 `mail.thread` 的模型)
- 在线客服对话——客服人员侧,通过 Discuss 频道视图访问

### 不在范围内

- **在线客服前端弹窗(访客侧)**。访客侧的弹窗组件与后台 composer
  不同。客服人员从后台 Discuss 视图处理对话,翻译功能在那里完全可用。
- **Knowledge 知识库文章**。Knowledge 模块的内容结构与
  `mail.thread` 不同,不在本模块覆盖范围内。
- **系统界面文字**(菜单、按钮、字段名)。这些由 Odoo 内置的
  i18n 翻译系统处理。

## AI 服务商扩展

默认版本支持 Odoo 原生 `ai` 模块集成的 AI 服务商:OpenAI 和
Google Gemini。模块的服务商层可扩展至其他 AI 服务商——例如:
默认服务商在特定区域不可达、特定语言对偏好使用领域专用模型、
或因组织规定需要使用其他服务商等情况。

如需咨询扩展其他 AI 服务商,请发邮件至 <Hello@suitestate.com>。

## 支持的语言

超过 100 种语言,包括:英语、阿拉伯语、简体中文、繁体中文、
西班牙语、印地语、葡萄牙语、孟加拉语、俄语、日语、德语、法语、
乌尔都语、印尼语、意大利语、土耳其语、韩语、越南语、波斯语、
波兰语、泰语、荷兰语、乌克兰语、希伯来语、斯瓦希里语等。
具体语言对的翻译质量取决于 AI 服务商。

从右到左书写的语言(阿拉伯语、波斯语、乌尔都语、希伯来语、
普什图语、库尔德语、意第绪语)在翻译预览中正确以 `dir="rtl"` 渲染。

## 系统要求

- Odoo 19 企业版
- 原生 `ai` 和 `ai_app` 模块(企业版自带)
- 在 *设置 → 通用设置 → AI* 中配置 OpenAI 或 Google Gemini 的
  API Key
- Odoo 服务器到 `api.openai.com` 和/或
  `generativelanguage.googleapis.com` 的网络出站访问

---

## 配置

1. 安装本模块。
2. 进入 *设置 → 通用设置 → AI*,至少添加一个服务商 API Key。
3. 在同一页面的 *Discuss AI Translate* 区域,打开
   **Enable Discuss AI Translate** 开关,选择默认翻译模型
   (默认 `gpt-5-mini`)。
4. 可选:设置 **Failover Model** 为另一服务商的模型。

完成后,每条消息上立刻出现 *AI Translate* 按钮,输入框上也会
出现翻译开关。

### 高级参数

两个系统参数可调整(*技术 → 参数 → 系统参数*):

| 参数                                       | 默认值  | 用途                       |
|--------------------------------------------|---------|----------------------------|
| `suite_ai_translate.max_input_chars`        | `8000`  | 单条消息翻译的字符数上限   |
| `suite_ai_translate.temperature`            | `0.2`   | LLM 采样温度               |

## 隐私

功能默认关闭。开启后,本模块仅传输:

- 待翻译的消息或输入框草稿的纯文本内容
- 目标语言代码

不传输的内容:作者/收件人姓名、合作伙伴记录、邮箱地址、电话号码、
附件、图片、文件、数据库记录、CRM 上下文、销售数据、发票数据,
也不向已配置的 AI 服务商之外的任何方发送任何分析或遥测数据。

翻译结果缓存在本地 Odoo 数据库中(`mail.message.translation` 表),
2 周未访问后自动清理。
